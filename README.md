# 1. Введение
Цель проекта — разработать комплексную онлайн-платформу по аренде автомобилей, которая объединит арендаторов и владельцев транспортных средств. Платформа будет состоять из 4 основных компонентов:
- **Основной сайт** - для клиентов, ищущих автомобили для аренды.
- **Личный кабинет партнера** - для владельцев автомобилей, предоставляющих их в аренду.
- **Административная панель** - для управления платформой и модерации.
- **CRM система** - для управления взаимоотношениями с клиентами и арендодателями, а также для внутреннего использования сотрудниками компании. 

# 2. Цели и задачи проекта
- Предоставить удобный интерфейс для поиска и бронирования автомобилей.
- Обеспечить владельцам автомобилей инструменты для управления своими транспортными средствами и бронированиями.
- Создать административную панель для модерации контента и управления системой.
- Реализовать CRM-функциональность для управления взаимодействием с клиентами и арендодателями.
- Обеспечить интеграцию с GPS-маяками для отслеживания автомобилей.

# 3. Функциональные требования

## 3.1 Основной сайт для клиентов

### 3.1.1 Регистрация и авторизация
- Регистрация пользователей через email и социальные сети.
- Восстановление пароля через email.
- Подтверждение контактных данных.
  
### 3.1.2 Поиск и фильтрация автомобилей
- Поиск по городу и району.
- Фильтрация по следующим параметрам автомобиля:
  - Марка 
  - Модель
  - Класс автомобиля: эконом, бизнес, премиум, комфорт.
  - Тип кузова.
  - Тип двигателя: бензин, дизель, электро, гибрид, газ.
  - Год выпуска.
  - Привод: передний, задний, полный.
  - Количество мест.
  - Коробка передач: автомат, механика.
  - Дополнительные опции (кондиционер, GPS, детское кресло и т.д.).
  - Страхование (ОСАГО, КАСКО)

### 3.1.3. Карточка автомобиля
- Отображение фотографий автомобиля.
- Подробное описание и характеристики.
- Условия аренды и стоимость.
- Отзывы и рейтинги от предыдущих арендаторов.
Информация о владельце (без персональных данных).

### 3.1.4. Бронирование автомобиля
- Выбор даты и времени аренды.
- Расчет стоимости аренды с учетом дополнительных услуг.
- Возможность добавления страховки и дополнительных опций.
- Онлайн-оплата через интегрированные платежные системы.
- Подтверждение бронирования через email и SMS.

### 3.1.5. Личный кабинет клиента
- Просмотр текущих и прошлых бронирований.
- Управление персональными данными.
- Возможность оставить отзыв о поездке.
- Просмотр уведомлений и сообщений.


## 3.2. Личный кабинет владельца автомобиля

### 3.2.1. Регистрация и верификация
- Регистрация владельцев с загрузкой документов для верификации.
- Модерация и подтверждение администраторами.

### 3.2.2. Управление транспортными средствами
- Добавление новых автомобилей с указанием:
  - Марки и модели.
  - Года выпуска.
  - VIN-номера.
  - Пробега.
  - Даты приобретения и стоимости.
  - Фотографий и описания.
- Редактирование и удаление информации об автомобиле.
- Установка доступности для бронирования (календарь).

### 3.2.3. Управление бронированиями
- Просмотр поступивших заявок.
- Подтверждение или отклонение бронирований.
- Общение с клиентами через внутренний мессенджер.
- Просмотр истории бронирований.
### 3.2.4. Финансовые операции
- Просмотр доходов от аренды по периодам.
- Запрос на вывод средств.
- Отчетность по платежам и комиссиям.
### 3.2.5. Статистика и аналитика
- Статистика по загрузке автомобилей.
- Отзывы и рейтинги клиентов.
- Анализ востребованности по сезонам.

## 3.3. Административная панель
### 3.3.1. Управление пользователями
- Просмотр и редактирование профилей клиентов и владельцев.
- Управление правами доступа.
- Блокировка и разблокировка аккаунтов.
### 3.3.2. Модерация контента
- Проверка и одобрение добавленных автомобилей.
- Модерация отзывов и комментариев.
- Управление категориями и справочниками.
### 3.3.3. Управление бронированиями
- Просмотр всех бронирований.
- Разрешение спорных ситуаций.
- Отмена бронирований в особых случаях.


## 3.4. CRM-система

### 3.4.1. Управление клиентами
- Хранение полной информации о клиентах и арендодателях:
  - Персональные данные.
  - История взаимодействий.
  - Предпочтения и сегментация.
- Возможность добавления заметок и комментариев к профилям пользователей.

### 3.4.2. Управление продажами и бронированиями
- Отслеживание всех стадий бронирования:
  - Запросы.
  - Подтверждения.
  - Оплаты.
  - Завершения аренды.
- Анализ конверсии и выявление узких мест в процессе бронирования.
- Инструменты для повышения продаж (скидки, акции, персональные предложения).
- 
### 3.4.3. Коммуникации
- Встроенный мессенджер для общения с клиентами и арендодателями.
- Интеграция с email и SMS-рассылками для массовых коммуникаций.
- Настройка шаблонов сообщений и автоматических уведомлений.
- 
### 3.4.4. Маркетинг и аналитика
- Сегментация базы данных для таргетированных кампаний.
- Управление маркетинговыми кампаниями и отслеживание их эффективности.
Интеграция с рекламными платформами (Google Ads, Facebook Ads).
Анализ поведения пользователей на сайте и в приложениях.

### 3.4.5. Отчеты и аналитика
- Генерация отчетов по различным метрикам:
  - Доходы и расходы.
  - Активность пользователей.
  - Эффективность маркетинговых кампаний.
- Визуализация данных (графики, диаграммы).
- Возможность экспорта отчетов в различные форматы (PDF, Excel).
- 
### 3.4.6. Интеграция с другими системами
- Синхронизация данных с административной панелью и другими компонентами платформы.


# 4. Нефункциональные требования

## 4.1. Производительность
- Время загрузки страниц не более 3 секунд при среднем подключении.
- Система должна выдерживать одновременную работу не менее 1000 пользователей.

## 4.2. Безопасность
- Шифрование данных пользователей (SSL/TLS).
- Защита от распространенных веб-атак (SQL-инъекции, XSS и др.).
- патчинг системы.

## 4.3. Масштабируемость
- Возможность горизонтального и вертикального масштабирования.
- Использование микросервисной архитектуры (при необходимости).

## 4.4. Доступность
- Гарантированная доступность сервиса на уровне 99.5%.
- Система резервного копирования и восстановления данных.

# 4.5. Юзабилити
- Интуитивно понятный интерфейс для всех типов пользователей.
- Адаптивный дизайн для корректного отображения на различных устройствах.
- Поддержка основных браузеров (Chrome, Firefox, Safari, Edge).

# 5. Архитектура системы
## 5.1. Клиентская часть (Frontend)
- Технологии: HTML5, CSS3, JavaScript.
- Фреймворк NextJS (для основного сайта)
- Фреймворк ReactJS (для личного кабинента партнера, админ панели и CRM системы)
- CSS-фреймворк: Tailwind CSS + Shadcn/UI.
- Особенности:
  - Адаптивный дизайн.
  - Поддержка SPA (Single Page Application) для быстрого отклика.
  - SSR для поисковых систем

## 5.2. Серверная часть (Backend)
- Язык программирования: NodeJS
- Фреймворк: Express.js или NestJS.
- База данных: PostgreSQL и MongoDB.
- API: RESTful API для взаимодействия между Frontend и Backend .
- Интеграции:
  - Платежные системы (Тинькофф).
  - Сервисы SMS и email-рассылок.
  - GPS-сервисы для отслеживания автомобилей.

## 5.3. Системная архитектура
- Серверы:
  - Использование облачных сервисов (AWS, Google Cloud, Azure) или выделенных серверов.
- Микросервисы (при необходимости):
  - Разделение функционала на независимые сервисы для улучшения масштабируемости и надежности.
- Кэширование:
  - Использование Redis для ускорения работы системы.
- Контейнеризация:
  - Использование Docker и Kubernetes для управления контейнерами и оркестрации.
